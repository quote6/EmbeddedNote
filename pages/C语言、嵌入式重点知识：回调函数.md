- 回调函数定义：
	- **回调函数**就是一个通过**函数指针**调用的函数。如果你把函数的指针（地址）作为参数传递给另一个函数，当这个指针被用来调用其所指向的函数时，我们就说这是回调函数。
	- 回调函数不是由该函数的实现方直接调用，而是在特定的事件或条件发生时由另外的一方调用的，用于对该事件或条件的响应。
- 关于`typedef`一些说明：
	- 通常的`typedef`取别名的方式为
	- ```C
	  typedef oldName NewName;
	  ```
	- 但是对于函数指针类型、数组类型定义取别名时需要区分，如：
	- ```C
	  typedef char Array20[20];
	  Array20 a1,a2; // 等价于 char a1[20], char a2[20]
	  ```
	- 关于数组类型，提一嘴，对于数据类型来说，数组长度也是类型的一部分，所以==不同的数组长度是不同的类型==，但是C语言存在隐式转换，这块显示的不明显。
	- `typedef`只是取别名，并不会创建新的类型，如：
	- ```C
	  #include <stdio.h>
	  typedef int MyInt;
	  int fun(MyInt num){
	    printf("%d\n",num++);
	    return 0;
	  }
	  
	  int main(){
	    int a = 0;
	    MyInt b = 10;
	    fun(a);
	    fun(b);
	    return 0;
	  }
	  ```
	- 上面的程序并不会报错，这说明`MyInt`和`int`仍为同一类型。
- 回调函数的作用本质上是解耦合，将通用的操作固定，将易变的操作分离出来。将程序结构拆分为地基与上层。
- 例如对于某个函数`data_print`，我们想让它可以打印不同的数据类型。通常的想法是传入数据的指针以及长度，然后函数内部使用`if`进行数据类型判断，来执行相应的数据打印。使用`if`语句可以发现，如果我们需求发生变化时，比如要让该函数支持打印一个新的数据类型，那么这个函数的`if`分支就需要继续添加。而以后的需求我们是难以预见的，因此想一劳永逸在`if`语句中列出所有分支是不可能的。这时候我们就可以想到，与其在函数内部判断不同类型，不如在给该函数传参的时候就已经完成相应分支的判断。因为函数的参数通常是会变化的，每次调用时，传给函数的参数通常都不一样，而函数体的代码通常是不应变化的。因此我们将易变性提升到函数参数上，使函数体内的代码固定。由用户传入如何显示的函数，我们根据再根据传入的具体数据类型调用用户传入的函数进行执行。这个用户传入的函数就可以认为是一个回调函数。
